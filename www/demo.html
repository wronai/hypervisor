taskinity.com · visual URI runtime demo

System agentowy wyjaśniony przez cele, grafy i workflow.

Ta paczka strony pokazuje nie tylko techniczne URI, ale przede wszystkim cel agenta: po co istnieje, co obserwuje, od czego zależy, co się stało i co użytkownik może zrobić dalej.

Runtime mówi, gdzie agent działa.

Cel wybranego agenta purpose

Weather Agent

Celem agenta pogodowego jest dostarczenie prognozy pogody jako capability. Dashboard nie pokazuje tylko procesu — tłumaczy, czy agent rzeczywiście działa.

Co obserwuje?runtime state, health endpoint, agent card i logi.
Co może zrobić?Dostarczyć prognozę albo zgłosić problem do repair supervisora.
Co oznacza błąd?Proces może istnieć, ale usługa HTTP może nie być gotowa.
Co dalej?Pokaż logi, uruchom diagnozę albo utwórz ticket.
Agent = kto? Cel = po co? Health = czy działa? Repair = co dalej?
